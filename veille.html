C'est quoi Symfony ?

Définition

Symfony est un framework PHP open-source créé en 2005 par Fabien Potencier chez SensioLabs. Il fournit un ensemble de composants réutilisables et d'outils pour construire des applications web robustes, performantes et maintenables.


C'est l'un des frameworks PHP les plus utilisés dans le monde professionnel, notamment en France, et il sert de base à d'autres projets majeurs comme Drupal, Laravel ou API Platform.

Architecture MVC

Symfony suit le patron de conception MVC (Modèle-Vue-Contrôleur) :

  • Modèle — gestion des données via Doctrine ORM
  • Vue — rendu HTML avec le moteur de templates Twig
  • Contrôleur — logique métier et traitement des requêtes
  • Routing — association des URLs aux contrôleurs
  • Services — injection de dépendances via le Container

Composants clés

  • Doctrine — ORM pour la base de données
  • Twig — moteur de templates sécurisé
  • Symfony Form — génération et validation de formulaires
  • Security — authentification, autorisation, guards
  • Messenger — files de messages asynchrones
  • Console — commandes CLI personnalisées

Pourquoi l'utiliser ?

  • Très utilisé en entreprise, notamment en France
  • Code structuré, maintenable et testable
  • Documentation complète et communauté active
  • Cycle de vie prévisible avec versions LTS
  • Compatible avec les dernières nouveautés PHP 8+
  • Suivi actif de la sécurité et des correctifs

Cycle de versions Symfony

Symfony suit un cycle prévisible : une version mineure tous les 6 mois (mai & novembre), une version majeure tous les 2 ans. Les versions LTS (X.4) bénéficient d'un support de 3 ans pour les bugs et 4 ans pour la sécurité. La prochaine LTS attendue est la 7.4 fin 2025.

Ce que j'ai appris en stage

Stage chez Enedis — Mai 2025 — Développeur Web PHP/Symfony (6 semaines)

  • Routing avec attributs PHP 8

    Utilisation des attributs #[Route] pour définir les routes directement dans les contrôleurs, sans fichier YAML.

  • Doctrine ORM

    Création d'entités, de repositories, et requêtes avec le QueryBuilder pour des listings filtrés et paginés.

  • Templates Twig

    Création de vues dynamiques avec héritage de templates, macros et filtres Twig pour les pages de listing MCO.

  • Checklist dynamique

    Développement d'un système de checklist MCO avec validation en temps réel, persistance en base et interface d'administration.

  • Méthode agile en équipe

    Travail en équipe sur plusieurs sites avec sprints, revues de code et gestion de tickets.
